Taller: Observabilidad y Gestión de Incidencias
Proyecto práctico enfocado en la implementación de un ciclo completo de gestión de incidencias y observabilidad de software en Python.

Objetivo
Aplicar técnicas de depuración, registro de eventos (logs) y gestión de incidencias en entornos colaborativos para evidenciar el flujo completo: Detección → Registro → Asignación → Resolución → Cierre.

Metodología
Detección: Identificación de errores y comportamiento anómalo mediante debugging en VS Code.

Registro: Implementación de niveles de logging (INFO, WARNING, ERROR, CRITICAL) y persistencia en app.log.

Gestión: Creación de tickets en Jira/Sentry con clasificación de severidad.

Matriz de Impacto: Clasificación técnica de incidencias (Críticas, Importantes, Moderadas, Bajas).

Resolución: Corrección del código base y actualización del estado del flujo de trabajo.

Tecnologías Utilizadas
Lenguaje: Python 3.x

Entorno: Visual Studio Code (Depuración activa)

Gestión de Proyectos: Jira / Sentry

Control de Versiones: Git / GitHub

Estructura del Repositorio
/src: Código fuente base y corregido.

/logs: Archivos de registro de eventos (app.log).

/evidencias: Capturas de consola, tracebacks y tableros de gestión.
